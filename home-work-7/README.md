# Домашка 7 - CSS позиционирование

Эта домашка оказалась интереснее чем я думал сначала. Вроде просто блоки двигать туда-сюда, но реально много нового понял.

## Что выучил

Разобрался с display свойством - block элементы идут друг под другом и занимают всю ширину, inline стоят рядом но им нельзя задать размер. Есть еще inline-block который берет лучшее от обоих - стоит рядом и можно задать ширину.

Box-sizing border-box это оказывается очень полезная штука. Без него когда задаешь width 300px на самом деле получается больше потому что padding и border добавляются сверху. С border-box все включается в эти 300px и не надо ничего высчитывать.

Float и clear - это старые методы для обтекания элементов. Сегодня уже не особо используется но для старых проектов полезно знать. Float делает элементы "плавающими" и контент обтекает их, а clear останавливает это обтекание.

Overflow нужен когда контент не помещается в контейнер. Лучше всего использовать auto - скроллбар появляется только если надо. Hidden просто обрезает все что не влезло.

Самое сложное это позиционирование. Там куча вариантов:

- static это обычное поведение по умолчанию
- relative сдвигается от своего места но оставляет пустоту
- absolute позиционируется относительно родителя с position (если нет такого то относительно body)
- fixed всегда на одном месте экрана даже когда скроллишь
- sticky сначала скроллится а потом прилипает

Главное что понял - для absolute обязательно надо родителю ставить position relative иначе элемент улетает не туда куда нужно.

Normalize и reset это две разные библиотеки. Normalize оставляет полезные стили браузера а reset вообще все убирает. Normalize удобнее потому что не надо все стилизовать с нуля.

В общем домашка была полезная, теперь понимаю как элементы двигаются и позиционируются на странице.
